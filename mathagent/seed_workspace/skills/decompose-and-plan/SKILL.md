# Decompose and plan

Before solving:
1. Identify the **givens**, the **goal**, and the **type** (compute a value, prove an
   identity/inequality, characterize all solutions, exists/for-all claim).
2. Write a short plan as an ordered list of sub-goals.
3. Choose a strategy explicitly: direct, induction, contradiction, contrapositive,
   case split, construction, invariant/monovariant, generating function, etc.
4. Only then start executing, one sub-goal at a time, checking each as you go.

A visible plan makes errors easy to localize and is what later verification hooks onto.
